# HabitLink 内测指南

本文档用于 P1 内测准备，帮助测试人员按统一流程验证 HabitLink MVP。当前内测版只覆盖已实现功能，不包含补打卡、TodoList、番茄钟、排行榜、提醒通知、微信登录、移动端应用和完整权限系统。

## 1. 内测目标

- 验证用户注册、登录、退出和 token 失效处理。
- 验证不同用户之间的数据隔离。
- 验证个人目标、今日打卡、防重复打卡和统计展示。
- 验证小组共同目标、邀请码加入、小组成员今日状态。
- 验证退出小组和转让组长的核心规则。
- 收集接口错误提示、页面文案和演示流程问题。

## 2. 测试环境

- 后端地址：`http://localhost:8080`
- 前端地址：通常为 `http://localhost:5173`
- 数据库：MySQL 8.x
- 浏览器：Chrome 或 Edge 最新稳定版

测试前建议确认：

- 后端使用 `dev` profile 正常启动。
- 前端 `.env.development` 中 `VITE_API_BASE_URL=http://localhost:8080`。
- 数据库已执行 `database/init.sql`。
- 如果是旧数据库，已按顺序执行 `database/migrations/` 下迁移脚本。

## 3. 测试账号

建议准备两个账号：

```text
student01 / 123456 / 小明
student02 / 123456 / 小红
```

`database/init.sql` 不内置账号，需要通过注册接口或前端页面创建。

## 4. 测试前清理

浏览器控制台执行：

```js
localStorage.clear()
```

如需重置数据，可以重新执行 `database/init.sql`。注意：该脚本会删除并重建相关表。

## 5. 推荐测试流程

1. 注册并登录 `student01`。
2. 创建一个个人目标。
3. 对个人目标完成今日打卡。
4. 查看目标统计中的 `totalDays` 和 `continuousDays`。
5. 创建一个小组，并填写小组共同目标。
6. 复制小组邀请码。
7. 退出登录。
8. 注册并登录 `student02`。
9. 使用邀请码加入小组。
10. 在目标列表查看小组目标。
11. 对小组目标打卡。
12. 回到小组页面查看成员今日状态。
13. 切回 `student01`，把组长转让给 `student02`。
14. `student01` 退出小组。
15. 验证 `student01` 自己的个人目标仍然可以删除。
16. 手动把 localStorage 中的 token 改成无效值，验证前端清理登录态并跳转登录页。

## 7. 通过标准

- 注册和登录流程可用。
- token 失效后能清理登录态并跳转登录页。
- `student02` 看不到 `student01` 的个人目标。
- 小组目标只在加入小组后可见。
- 小组成员今日状态只基于该小组绑定目标判断。
- 小组目标不能在目标页直接删除。
- OWNER 多人小组退出前必须先转让组长。
- OWNER 单人小组退出后会删除空小组和绑定目标。
