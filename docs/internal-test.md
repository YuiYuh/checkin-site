# HabitLink 内测说明

本文档用于小范围内测和课程演示前检查。

## 1. 内测目标

- 验证核心业务流程能完整走通。
- 验证用户隔离、目标打卡、小组共同目标、退出小组和转让组长逻辑。
- 验证 token 失效后的前端体验。
- 收集页面文案、交互和接口错误提示问题。

## 2. 测试范围

本轮包含：

- 注册和登录
- 个人目标
- 今日打卡
- 累计和连续天数统计
- 小组创建和邀请码加入
- 小组共同目标
- 小组成员今日目标完成状态
- 退出小组
- 转让组长
- token 失效处理

本轮不包含：

- 补打卡
- TodoList
- 番茄钟
- 排行榜
- 提醒通知
- 微信登录
- 移动端
- 完整生产权限系统

## 3. 测试账号建议

建议准备两个账号：

```text
student01 / 123456 / 小明
student02 / 123456 / 小红
```

`database/init.sql` 不内置账号，需要通过注册接口或前端注册入口创建。

## 4. 测试前准备

1. 初始化数据库或确认迁移已执行。
2. 启动后端。
3. 启动前端。
4. 浏览器打开前端地址。
5. 清空 localStorage，避免旧 token 干扰。

清空 localStorage 可在浏览器控制台执行：

```js
localStorage.clear()
```

## 5. 推荐内测流程

1. 注册并登录 `student01`。
2. 创建个人目标。
3. 对个人目标完成今日打卡。
4. 查看统计。
5. 创建小组，并填写小组共同目标。
6. 复制邀请码。
7. 退出登录。
8. 注册并登录 `student02`。
9. 使用邀请码加入小组。
10. 查看目标列表中的小组目标。
11. 对小组目标打卡。
12. 查看小组成员今日状态。
13. 使用 `student01` 转让组长给 `student02`。
14. `student01` 退出小组。
15. 验证 `student01` 的个人目标仍可删除。
16. 手动篡改 localStorage token，验证自动跳转登录页。

## 6. 缺陷记录模板

```text

环境：

实际结果：
预期结果：
截图或接口响应：
严重程度：
备注：
```

## 7. 通过标准

- 关键流程无阻塞问题。
- 两个用户数据不会互相泄露。
- 小组共同目标状态按 `team.goal_id` 判断。
- 小组目标不能在目标页直接删除。
- OWNER 多人小组退出必须先转让组长。
- token 失效后能清理登录态并跳转登录页。
