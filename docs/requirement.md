# HabitLink 需求说明

## 项目背景

大学生在课程学习、英语复习、运动锻炼和小组协作中，经常需要通过持续打卡来保持习惯。现有工具通常功能较重，或偏向个人记录，缺少适合课程小组作业展示的轻量级协作场景。

HabitLink 面向大学生和学习小组，提供简单的目标创建、每日打卡、打卡统计和小组监督能力。MVP 阶段优先完成核心业务闭环，本需求文档只约定当前 MVP 的实现范围。

## 项目目标

- 完成一个可演示的打卡网站 MVP
- 支持个人目标打卡和基础统计
- 支持学习小组通过邀请码加入
- 支持查看小组成员今日打卡状态
- 在数据库中预留必要扩展字段，但不实现微信登录、移动端或线上部署功能

## 用户角色

### 普通用户

- 注册并登录系统
- 创建个人打卡目标
- 每日完成打卡
- 查看个人打卡记录和统计
- 加入学习小组

### 小组创建者

- 创建学习小组
- 分享邀请码
- 查看小组成员
- 查看成员今日打卡状态

## MVP 功能需求

### 用户注册登录

- 用户可以通过用户名和密码注册
- 用户可以通过用户名和密码登录
- 系统保存用户昵称、头像等基础资料
- 数据库预留 `avatar` 和 `openid` 字段，方便后续支持微信小程序登录

### 创建打卡目标

- 用户登录后可以创建打卡目标
- 目标包含标题、描述、开始日期和结束日期
- 目标默认状态为进行中

### 查看目标列表

- 用户可以查看自己创建的目标列表
- 列表展示目标名称、状态、创建时间和今日是否打卡

### 今日打卡

- 用户可以对某个目标进行今日打卡
- 打卡可以填写简单备注
- 每个用户对同一目标每天只能打卡一次

### 防重复打卡

- 后端需要校验重复打卡
- 数据库 `checkin_record` 表通过 `user_id`、`goal_id`、`checkin_date` 唯一约束兜底
- 重复打卡时返回明确错误信息

### 查看打卡记录

- 用户可以查看指定目标的打卡记录
- 支持按日期范围查询

### 统计累计打卡天数

- 系统统计某个目标下用户的累计打卡天数
- 用于展示学习投入和习惯养成结果

### 统计连续打卡天数

- 系统从最近日期向前计算连续打卡天数
- 若今日未打卡，可从昨日开始计算，具体规则可在实现阶段确定

### 创建小组

- 用户可以创建学习小组
- 系统生成唯一邀请码
- 创建者自动成为小组成员，角色为 `owner`

### 邀请码加入小组

- 用户输入邀请码加入小组
- 同一用户不能重复加入同一小组

### 查看小组成员

- 小组成员可以查看组内成员列表
- 展示成员昵称、头像和加入时间

### 查看小组成员今日是否打卡

- 小组成员可以查看组内成员今日打卡状态
- MVP 阶段可先统计成员今日是否存在任意打卡记录
- 后续可扩展为按指定目标或小组目标统计

## 非功能需求

- 接口统一以 `/api` 开头
- 接口统一返回 JSON 格式
- 后端使用分层结构，便于后续扩展
- 数据库字段使用 `created_at` 和 `updated_at` 记录时间
- 前端页面保持简洁，适合课程演示和小组协作场景
- 代码和文档应便于四人并行开发

## 边界与暂不实现内容

- MVP 阶段不实现真实微信登录，只预留 `openid` 字段
- MVP 阶段不实现复杂权限管理
- MVP 阶段不实现消息提醒、排行榜和文件上传
- MVP 阶段不实现生产环境部署脚本，只保留运行方式占位

## 范围外说明

- 微信小程序端登录与打卡不属于当前 MVP
- 小组目标、组内排行榜、打卡提醒和通知不属于当前 MVP
- 管理后台和数据看板不属于当前 MVP
- 云服务器部署、HTTPS 和域名访问不属于当前 MVP
